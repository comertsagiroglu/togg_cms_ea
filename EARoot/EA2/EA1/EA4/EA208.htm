Pre-conditions:<br/>- Camera serializer and Display deserializer are locked.<br/>- Image Sensor is in streaming state<br/>- Power state = POWER_ON<br/>- Operational state = PRIMARY_OPERATION<br/><br/>Scenario:<br/>- SYS_CHECK output of the Image Sensor is connected to an external interrupt input of the camera MCU<br/>- When image orientation failure is detected on the Image Sensor, SYS_CHECK signal is triggered and this generates interrupt event on the camera MCU.<br/>- Image sensor sets the TBD register (TBD:image orientation fault related as149 register)<br/>- In the SYS_CHECK interrupt routine camera status SYS_CHECK bit is set<br/>- Display queries the camera status byte every 100 msec.<br/>- When the display receives the SYS_CHECK bit of the camera status byte set, it transitions to the safe state at which the LCD backlight is disabled.<br/>- Display Operational state is set to CMS_FAILURE and returned to IDCC when it is queried.<br/>- Display reads the safety related Image Sensor registers and determines the fault which triggered the SYS_CHECK event and sets the related diagnostic code.<br/>- IDCC  diagnostic data query period is  100msec.<br/>- IDCC  operational mode query period is  100msec.<br/><br/>Post-conditions:<br/>Operational state = CMS_FAILURE<br/>LCD backlight off<br/><br/>
