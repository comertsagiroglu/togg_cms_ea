Pre-conditions:<br/>- Camera serializer and Display deserializer are locked.<br/>- Image Sensor is in streaming state<br/>- Power state = POWER_ON<br/>- Operational state = PRIMARY_OPERATION<br/>- LCD is functional and has no active failure<br/><br/>Scenario:<br/>- Display device check function reads the LCD failure flags throught the SPI interface.<br/>- LCD failure flags:<br/>    LVDS lock failure<br/>    GAS function failure<br/>    Interal source circuit failure<br/>    Tradition gate signal failure<br/>    Self-protection mode failure<br/>    EEPROM reload failure<br/>    OTP trimming failure<br/>    PFM abnormal failure<br/>    Glass crack failure <br/>    OTP reload failure<br/>- When an LCD device failure is detected, lcd failure event is generated.<br/>- LCD error handler sets the operational state to CMS_FAILURE and returns it to IDCC when queried.<br/>- LCD error handler disables the LCD backlight<br/>- Display fw sets the related diagnostic code and returns it to IDCC when queried.<br/>- Display device check period is 100msec<br/>- IDCC  diagnostic data query period is  100msec.<br/>- IDCC  operational mode query period is  100msec.<br/><br/>Post-conditions:<br/>Operational state = CMS_FAILURE<br/>LCD backlight off<br/><br/>
