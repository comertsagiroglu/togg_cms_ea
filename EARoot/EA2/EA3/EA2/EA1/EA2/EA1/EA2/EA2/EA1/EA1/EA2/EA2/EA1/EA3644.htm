This function enables that. <br/>@param InstancePtr is the pointer to the DMA engine to work on<br/>@param BdAddrPhys is the physical starting address for BDs<br/>@param BdAddrVirt is the Virtual starting address for BDs. For systems that do not use MMU, then virtual address is the same as physical address<br/>@param NumBds is the number of BDs to setup with. This is required to be the same as the number of frame stores for that channel<br/>@param Direction is the channel direction <br/>@return - XST_SUCCESS for a successful setup - XST_DEVICE_BUSY if the DMA channel is not idle, BDs are still being used - XST_INVALID_PARAM if parameters not valid - XST_DEVICE_NOT_FOUND if the channel is invalid <br/>@notes We assume that the memory region starting from BdAddrPhys and BdAddrVirt are large enough to hold all the BDs.<br/>
