Notes: portYIELD_FROM_ISR() does not directly call vTaskSwitchContext(), but instead sets a flag to say that a yield has been requested.  The interrupt exit code then checks this flag, and calls vTaskSwitchContext() before restoring a task context, if the flag is not false.  This is done to prevent multiple calls to vTaskSwitchContext() being made from a single interrupt, as a single interrupt can result in multiple peripherals being serviced.<br/>
