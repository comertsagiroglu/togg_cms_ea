WDT_Clear():void
Details:
Sequential
Notes: If application intends to stay in active mode after clearing WDT, then use WDT_Clear API to clear the WDT. This avoids CPU from waiting or stalling for Synchronization. If application intends to enter low power mode after clearing WDT, then use the WDT_ClearWithSync API to clear the WDT.<br/>

Details:
Sequential
Notes: This API must be used if application intends to enter low power mode after clearing WDT. It waits for write synchronization to complete as the device must not enter low power mode while write sync is in progress.<br/>
